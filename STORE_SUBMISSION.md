### 7. Submission Commands

#### Build and Submit to App Stores
```bash

# iOS

```

```bash
make eas-build-all ENV=production
```

### Version Updates
When releasing updates:
1. Update version in `app.json`:
   - `version`: "1.0.1"
   - iOS `buildNumber`: "2"
   - Android `versionCode`: 2
2. Update `CHANGELOG.md`
3. Build and submit:
   ```bash
   make eas-build-all ENV=production
   ```

## üí° Tips

1. **Start with TestFlight/Internal Testing**: Don't submit to production immediately
2. **Privacy Policy is Mandatory**: Both stores reject apps without one
3. **Screenshots Sell Apps**: Invest time in good screenshots
4. **Use Staged Rollout**: Google Play allows releasing to small % of users first
5. **Monitor Analytics**: Track user acquisition and retention post-launch
6. **Respond to Reviews**: Good for rankings and user satisfaction
